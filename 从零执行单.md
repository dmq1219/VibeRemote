# VibeRemote 从零执行单（修订版）

> 原则：每阶段必须通过验收后再进入下一阶段；先验证输入链路，再做映射和 UI。
> 修订要点：多输入设备架构（小米遥控器 + PS4/PS5/PS3 手柄）、日志必须无窗口可取证、
> 签名身份稳定 = TCC 权限稳定、切换 App 后自动聚焦输入框。

## 0. 固定范围与身份

- [ ] App 名称固定为 `VibeRemote`
- [ ] Bundle Identifier 从第一天起固定，不再更换
- [ ] 固定唯一代码签名身份，确认钥匙串中同时存在证书和私钥
- [ ] 执行 `security find-identity -v -p codesigning` 验证签名身份
- [ ] ⚠️ 签名身份稳定是 TCC 权限（输入监控/辅助功能）稳定的前提：
      ad-hoc 或 Sign to Run Locally 每次重签都会作废已授权条目
- [ ] 最低系统版本固定为 macOS 13
- [ ] 支持设备清单（从第一天定义，映射层与设备解耦）：
  - 小米语音遥控器：Vendor `0x2717`，Product `0x32B0`（主线，原始 HID）
  - PS5 DualSense：`0x054C/0x0CE6`（GameController 框架）
  - PS4 DualShock 4：`0x054C/0x05C4`、`0x054C/0x09CC`（GameController 框架）
  - PS3 DualShock 3：`0x054C/0x0268`（实验性，仅 USB，原始 HID）
- [ ] 定义统一的设备无关按键事件 `ButtonEvent`（如 `.up/.down/.confirm/.back/.home/.menu/...`），
      映射层、状态机、Overlay 只消费 ButtonEvent，不直接消费 HID 码或手柄按键
- [ ] 不加入按应用映射（保持全局映射）
- [ ] 正式运行路径固定为 `/Applications/VibeRemote.app`
- [ ] 不在系统中同时保留多个可搜索的 `.app` 副本

验收：

- [ ] Bundle ID、签名身份、Team ID、安装路径全部确定
- [ ] 不再使用缺少私钥的证书
- [ ] 不在 Apple Development 与 Sign to Run Locally 之间反复切换
- [ ] ButtonEvent 枚举定稿，四类设备都能映射到它

## 2. 先完成原始 HID 监听（小米遥控器）

执行顺序：

- [ ] `IOHIDManagerCreate`
- [ ] `IOHIDManagerSetDeviceMatchingMultiple`
- [ ] 注册 Device Matching Callback
- [ ] 注册 Device Removal Callback
- [ ] `IOHIDManagerRegisterInputValueCallback`
- [ ] `IOHIDManagerRegisterInputReportCallback`
- [ ] `IOHIDManagerScheduleWithRunLoop`
- [ ] `IOHIDManagerOpen`

匹配条件：

- [ ] 匹配表来自 §0 设备清单（小米 + PS3），不硬编码在回调里
- [ ] 找到后输出 `Connected <设备名>`

错误日志：

- [ ] 每个有返回值的调用输出十六进制 `IOReturn` + 可读原因
- [ ] `0xE00002E2` 明确显示为权限或设备访问策略拒绝
- [ ] void API 只记录“已注册”，不伪造返回码

回调策略：

- [ ] Manager 打开成功时，只使用 Manager 的 Input Value/Input Report callbacks
- [ ] 此时禁止再给同一设备注册直接 Device callbacks
- [ ] Manager 打开失败时才使用 Device 级 fallback（Open/Register/Schedule）
- [ ] Manager 与 Device 两套回调必须互斥
- [ ] 设备回调只接受已匹配设备
- [ ] Report buffer 生命周期覆盖整个监听周期
- [ ] 退出时关闭、取消调度并释放 buffer

验收：

- [ ] 每按一次按键，只出现一次 Report 映射动作
- [ ] 不出现两个 Overlay、不出现同一个 `[Mapping]` 两次
- [ ] 断开、重连遥控器后状态正确恢复

## 2b. PS4 / PS5 / PS3 手柄支持

技术选型（重要，不要全走原始 HID）：

- [ ] PS4 DualShock 4、PS5 DualSense 使用 Apple `GameController` 框架（`GCController`）：
  - 系统原生支持蓝牙与 USB 连接、按键语义、电量
  - 不需要输入监控权限，代码量远小于原始 HID
- [ ] 监听 `GCController.didConnectNotification` / `didDisconnectNotification`
- [ ] 使用 `extendedGamepad` profile 读取按键
- [ ] PS3 DualShock 3 标记为实验性：
  - GameController 框架不支持；蓝牙配对在现代 macOS 上不可行，**仅支持 USB 直连**
  - 走原始 HID（复用 §2 的 Manager），匹配 `0x054C/0x0268`
  - 连接后必须先发送 Feature Report `0xF4`（enable operational mode），否则不上报按键
  - PS3 失败不阻塞主线，UI 中标注“实验性，仅 USB”

按键 → ButtonEvent 默认映射：

| 手柄按键 | ButtonEvent |
|---|---|
| 方向键 上/下/左/右 | up / down / left / right |
| ×（cross） | confirm（确认） |
| ○（circle） | back（返回） |
| PS 键 | home（应用切换器 / 长按听写） |
| □（square） | menu（Fn 听写切换） |
| △（triangle） | 可配置，默认 ignore |
| L1 / R1 | 音量−（Cmd+[）/ 音量+（Cmd+]） |
| 其余按键、摇杆 | 默认 ignore，可在设置中分配 |

要求：

- [ ] 手柄事件统一转换为 ButtonEvent 后走与遥控器完全相同的映射层、状态机、Overlay
- [ ] 手柄与遥控器可同时连接，事件互不干扰
- [ ] 手柄断开/重连自动恢复
- [ ] 设置界面显示手柄连接状态与型号（DualSense / DualShock 4 / DualShock 3）

验收：

- [ ] PS5 手柄蓝牙配对后，方向键/×/○/PS 键与遥控器行为一致
- [ ] PS4 手柄同上
- [ ] PS 键短按打开应用切换器，长按听写
- [ ] □ 键可开始/停止听写并写入文字
- [ ] PS3 手柄 USB 直连可用（实验性验收，允许失败但要有明确日志）

工具：

| 按键 | HID Code |
|---|---:|

## 4. 权限与签名

首次运行提示两项独立权限：

- [ ] 输入监控：用于读取 HID 输入（GameController 路径不需要）
- [ ] 辅助功能：用于发送 CGEvent、聚焦目标 App 输入框

授权检查：

- [ ] `IOHIDCheckAccess` / 未决定时 `IOHIDRequestAccess`
- [ ] `AXIsProcessTrusted` / 需要时 `AXIsProcessTrustedWithOptions` 弹出提示
- [ ] 权限变化后提示用户退出并重新启动

必须核对：

- [ ] 当前进程绝对路径是 `/Applications/VibeRemote.app`
- [ ] Bundle Identifier、Code Signature 与授权条目一致
- [ ] 授权的不是 DerivedData、Debug 或 outputs 中的副本

⚠️ 二进制替换规则（血泪教训）：

- [ ] 每次向 `/Applications` 覆盖安装新构建后，立即重新验证两项权限
- [ ] 签名身份不稳定（ad-hoc）时，重签会作废 TCC 授权——表现为“昨天还好好的，今天没反应”
- [ ] 权限异常的第一反应是核对签名与授权条目，不是改代码

遇到 `0xE00002E2` 的排查顺序：

1. 运行路径 → 2. Bundle ID → 3. 签名与私钥 → 4. 输入监控授权 → 5. 完全退出重开。
不先修改 HID 解析代码。

验收：

- [ ] `IOHIDManagerOpen: 0x00000000 - success`
- [ ] `AXIsProcessTrusted()` 返回 true
- [ ] 重建 Release 并覆盖安装后，权限身份仍稳定

## 5. CGEvent 与听写

普通快捷键：

- [ ] Key Down + Key Up，正确虚拟键码
- [ ] 组合键在按下和松开阶段携带相应修饰键
- [ ] 使用 `.cghidEventTap` 发送；发送前检查辅助功能权限

Fn 特殊规则：

- [ ] Fn 虚拟键码使用 `63`
- [ ] Fn Key Down：flags 包含 `.maskSecondaryFn`
- [ ] Fn Key Up：flags 必须为空（合成事件要模拟真实键盘的完整状态变化）
- [ ] 每次按键只发送一个完整的 Fn 按下/松开周期

听写：

- [ ] HOME 长按触发一次 Fn；菜单键 `0x65`（手柄 □ 键）触发一次 Fn
- [ ] 第一次 Fn 开始系统听写，第二次停止并把文字写入当前输入框
- [ ] 听写切换防抖：Fn 发出后 1 秒内忽略重复的听写触发
- [ ] 不调用 Siri；不强制修改 CoreAudio 默认输入设备
- [ ] 出现麦克风异常时，先确认 HID 和 Fn 事件数量，不先猜麦克风路由

验收：

- [ ] 实体 Mac Fn 的行为与遥控器菜单键一致
- [ ] 开始和停止各只需按一次；不出现 Fn 卡在按下状态
- [ ] 不出现一次实体按键发送两次 Fn

## 13. “没反应”故障分流

### A. 没有任何启动日志
- [ ] 用 `log show` 而不是 stdout 取证（print 会被缓冲）
- [ ] 确认运行二进制路径、进程启动时间晚于构建时间
- [ ] 确认监听在 App 启动时调用（不是窗口 onAppear）

### B. 有启动日志，但 Manager Open 失败
- [ ] 查 `IOReturn` → 输入监控 → 签名/Bundle ID/路径 → 授权条目对应当前副本
- [ ] 想起 §4：二进制刚替换过？先查 TCC

### C. 显示 Connected，但没有事件
- [ ] callback 注册顺序、RunLoop 调度、sender 归属、Manager/Device 互斥

### D. HID 和 Overlay 正常，但没有快捷键
- [ ] Enable Mapping → 辅助功能 → keycode/flags → 目标 App 是否接受

### E. Fn 没反应
- [ ] Key Down 带 `.maskSecondaryFn`、Key Up flags 为空、一次按键一个周期、系统听写设置

### F. 一次按键出现两次动作
- [ ] Manager/Device 双注册、50ms 防重复

### G. 开发时正常，重新登录后失效
- [ ] 初始化是否挂在窗口 `.onAppear`；无窗口冷启动测试；Launch at Login 验收

### H. 切换 App 后光标不在输入框
- [ ] 辅助功能权限是否有效（重装后可能作废）
- [ ] Electron 应用是否已设 `AXManualAccessibility`
- [ ] AX 搜索是否命中（角色识别/兜底规则）
- [ ] 设焦点失败时是否触发了点击兜底

### I. 手柄没反应
- [ ] PS4/PS5：`GCController.controllers()` 是否有设备；蓝牙是否配对到本机
- [ ] PS3：是否 USB 直连；是否发送了 `0xF4` enable report
- [ ] 手柄事件是否正确转换为 ButtonEvent
